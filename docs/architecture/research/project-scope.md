# 项目目标与约束

最后整理时间：2026-03-25

## 项目目标

本项目的目标不是做外部内存修改器，也不是做联机作弊工具，而是：

- 以 `BepInEx` 插件形式给《Enter the Gungeon》添加本地 Mod
- 在“新 run 开始”或“进入地牢后的合适时机”随机给玩家一组基础游戏已有物品
- 随机对象优先限定为：
  - 枪械
  - 被动道具
  - 主动道具

## 第一阶段明确不做的内容

- 不做外部内存扫描、CE 表、AOB、Trainer
- 不做联机相关逻辑
- 不做自定义美术、AssetBundle、音频
- 不做自定义枪弹、复杂 projectile 行为
- 不优先做游戏内配置 UI

## 第一阶段真正要解决的问题

第一版只需要解决两个核心问题：

1. 在什么时机执行“发开局物品”最稳妥
2. 用什么调用链把基础游戏已有 `PickupObject` 正确发给当前玩家

## 项目边界判断

对于这个项目，更合适的路线是“Mod 插件”而不是“传统修改器”：

- 原因 1：ETG 已有成熟的 Unity / C# 模组生态
- 原因 2：发放基础游戏物品属于游戏逻辑层操作，不需要外部注入工具链
- 原因 3：现成资料、示例项目和 API 都围绕 `BepInEx` 与 `ModTheGungeonAPI`

## 当前阶段的实现风格

第一版建议保持最小化：

- 小型单 DLL 插件
- 小规模白名单随机池
- 强调稳定触发与可复现日志
- 暂不追求大而全的功能集
