## [v0.2.3] - 2026-04-14

### 新增功能 (Added)
* 新增 `Boss Rush` 模式运行时实现，加入 `src/RandomLoadout/Runtime/BossRushService*.cs`、`BossRushHooks.cs`、`BossRushEncounter.cs` 与 `BossRushState.cs`，支持固定楼层顺序、Boss 房接管、奖励后自然乘电梯推进、失败或手动退出返回选人大厅。
* 新增 `F7` 面板中的 `Boss Rush` 页面与操作入口，加入 `src/RandomLoadout/Commands/InGameCommandController.BossRush.cs`，支持开始对战、查看当前楼层与状态、手动返回大厅。

### 技术改进/重构 (Changed/Refactor)
* 接入 `ModTheGungeonAPI` 与 `0Harmony` 作为 Boss Rush 运行时依赖，并更新 `src/RandomLoadout/Plugin.cs`、`src/RandomLoadout/RandomLoadout.csproj` 与 `Plugin*.cs` 启动链路。
* 重构 Boss Rush 运行时代码为职责拆分的 partial 结构，按流程、房间扫描、玩家初始化、日志拆分 `BossRushService`，降低单文件复杂度。
* 统一 character-select-hub 语义命名，将 `Breach` 相关运行时命名对齐为 `CharacterSelect` / `character select hub`，覆盖 `Plugin.State.cs`、`RunLifecycleTracker.cs`、`RunSceneWatcher.cs` 等运行时路径。

### 修复 (Fixed)
* 修复 Boss Rush 从大厅进入楼层时角色缺枪、无法装备与无法开火的问题，补齐 `Foyer.Instance.OnDepartedFoyer()`、首层玩家 bootstrap、传送后输入恢复与相机控制复位。
* 修复 Boss Rush 进入错误房间的问题，改为优先识别 `Boss Foyer`，仅在找不到前厅时回退到 `RoomCategory.BOSS`，避免误传送到商店或普通战斗房。
* 修复 Boss Rush 击败 Boss 后强制切层造成的流程错乱，改为保留原版奖励与电梯推进，只在观察到下一层场景后继续接管。
* 修复 Boss Rush 返回大厅或死亡回大厅后状态未清理的问题，新增 character-select-hub 状态判定与兜底强制 `Reset()`，使模式可重复开启。
* 修复 Boss Rush 的 Harmony Hook 稳定性问题，增加目标方法签名校验、游戏结束与暂停退出拦截，并修正返回选人大厅链路。

### 弃用/删除 (Deprecated/Removed)
* 删除 Boss Rush 中“Boss 已击败”“领取奖励后继续”“正在加载下一层”这类遗留运行时状态提示及对应的中英文文案。
