## [v0.2.1] - 2026-04-10

### 新增功能 (Added)
* 新增命令面板的裸输入发放能力，支持直接输入 `pickupId`、`alias`、`internalName` 或兼容的 `displayName`，无需先写 `gun`、`passive`、`active` 或 `item`。
* 新增 `Pickups` 图形化物品选择器页面，支持小图标列表、搜索、`All/Gun/Passive/Active` 分类筛选，以及点击结果直接发放。
* 新增运行时物品图标复用链路，直接读取游戏内 `PickupObject` 的 live sprite 数据绘制浏览页图标，不再依赖额外打包的图标资源。
* 新增 `src/RandomLoadout/Commands/GrantCommandService.cs` 的 catalog-entry 发放入口，使图形化物品选择器可以绕过字符串解析直接按选中物品发放。

### 技术改进/重构 (Changed/Refactor)
* 调整 `src/RandomLoadout.Core/Commands/GrantCommandParser.cs` 的解析策略，已知前缀仍按原类别命令处理，未知前缀与纯裸输入统一按 `Any` 解析。
* 重构 `src/RandomLoadout/Commands/InGameCommandController.cs` 的 F7 面板结构，在保留原文本输入区的前提下新增 `Pickups` 浏览页，并统一收口搜索、筛选、结果列表与直接发放交互。
* 调整 `src/RandomLoadout/Plugin.cs` 的命令控制器装配方式，额外注入可发放 catalog 与 alias 注册表，供图形化物品选择器复用。
* 调整 `docs/reference/commands.md` 的命令说明，明确所有查询输入按大小写不敏感处理，并将推荐示例统一为小写 `alias`、`internalName` 或 `pickupId`。

### 修复 (Fixed)
* 修复输入不以已知前缀开头时直接报 `Unknown command target` 的问题，改为将整句输入按物品查询处理。
* 修复需要快速发放单个物品时必须先补写类别前缀的问题，同时保留单独输入 `gun`、`passive`、`active`、`item` 时的缺少物品名提示。
* 修复 `src/RandomLoadout/Etg/EtgPickupResolver.cs` 在 `internalName` 未命中、`displayName` 实际发生重名时仍错误返回 `No pickup matched` 的问题，改为正确返回显示名歧义结果。
* 修复 `src/RandomLoadout/Commands/GrantCommandService.cs` 的重名提示文案，使 `gun Casey` 这类输入在可消歧时优先提示 `casey_bat`、`casey_nail` 这类可直接使用的 alias，而不是仅给出泛化建议。
