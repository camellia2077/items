## [v0.2.0] - 2026-04-09

### 新增功能 (Added)
* 新增更接近 `ModTheGungeonAPI give` 的物品输入方式，支持在命令面板与规则字符串中优先使用 `internalName`，例如 `passive PlatinumBullets`。
* 新增更细的物品发放结果日志，在命令结果与 `[RandomLoadout][Grant]` 日志中明确标记 `Path=primary` 或 `Path=fallback`，用于区分主路径与兜底路径。

### 技术改进/重构 (Changed/Refactor)
* 调整物品解析顺序为 `id -> alias -> internalName -> displayName`，统一命令面板、`specific.name` 与 `random.pool` 的字符串解析语义。
* 调整物品发放主路径，优先使用 `LootEngine.TryGivePrefabToPlayer(...)`，并保留分类专用 grant 作为兼容兜底。
* 调整 `docs/reference/commands.md` 与 `docs/reference/pickups.md` 的输入说明，改为推荐 `internalName` / `pickupId`，弱化 `displayName` 的优先口径。
* 重构 `scripts/generate_pickup_docs.py` 的别名读取链路，使其正确支持 `RandomLoadout.aliases.json5`。

### 修复 (Fixed)
* 修复 `Rapid OFF / Rapid ON` 会影响玩家全部枪械的问题，改为每把枪独立记录开关状态，只对当前枪生效。
* 修复 `Marine` 在 Breach `Switch Only` 路径下无法切换的问题，改为按当前环境中的 `marine` prefab 名加载角色。
* 修复部分物品只能通过显示名不稳定匹配的问题，改为优先按更稳定的 `internalName` / `pickupId` 解析。
* 修复 `scripts/generate_pickup_docs.py` 将 `RandomLoadout.aliases.json5` 当作纯 JSON 读取而导致文档生成失败的问题。
