## [v0.1.1] - 2026-04-05

> Note: this entry reflects the repository structure and file names used at the time of `v0.1.1`. Current file names may use newer `json5`-based naming and updated `docs/` paths.

### 新增功能 (Added)
* 新增 `RandomLoadout.aliases.json` 全局别名注册表，支持规则文件与游戏内命令共用 `alias -> pickupId` 可读引用。
* 新增 `RandomLoadout.pickups.json`、`RandomLoadout.pickups.by-category.json` 与 `RandomLoadout.rules.full-pool.json` 导出产物，为全物品随机池与指定物品输入提供统一索引。
* 新增 `defaults/catalog/` 仓库基线快照，并新增 `scripts/generate_pickup_docs.py`，可从 `defaults/catalog/RandomLoadout.pickups.by-category.json` 生成 `docs/pickups.md` 用户向物品手册。
* 新增 Breach 角色切换页，在 `F7` 命令面板中通过 `Characters` 入口单独查看与切换角色。
* 新增固定角色表驱动的角色选择 UI，在原生 Breach 角色旗子缺失时仍显示 `Marine`、`Hunter`、`Pilot`、`Convict`、`Robot`、`Bullet`、`Paradox`、`Gunslinger`。
* 新增隐藏角色点击解锁流程，支持在角色页中对 `Robot`、`Bullet`、`Paradox`、`Gunslinger` 持久解锁并立即写入存档。
* 新增 `Full Ammo` 调试按钮，支持一键补满当前装备枪械的总弹药与弹夹。
* 新增 `Rapid OFF / Rapid ON` 开关按钮，支持在命令面板内一键切换“按住左键按最快连点速率持续射击”的临时快速射击模式。

### 技术改进/重构 (Changed/Refactor)
* 重构规则加载链路，统一采用 `RandomLoadout.rules.json` -> `RandomLoadout.rules.full-pool.json` -> 内置应急默认 的三级回退顺序。
* 调整 `scripts/deploy_mod.py` 与 `scripts/script_common.py`，将部署同步范围从默认配置扩展到仓库默认文件与物品索引快照。
* 调整文档结构，在 `README.md` 与 `docs/README.md` 中补充默认快照、玩家物品手册与最新历史入口。
* 重构 Breach 角色枚举链路，优先复用原生 `Foyer.SetUpCharacterCallbacks()`，并在此基础上合并已知角色表以降低场景对象组织差异带来的影响。
* 调整命令面板布局，将角色切换从主面板底部按钮区拆分为独立页面，并统一当前角色、锁定角色与待切换状态的显示逻辑。
* 调整 `src/RandomLoadout/Commands/InGameCommandController.cs` 的角色页数据获取策略，仅在 `Characters` 页面执行角色查询并增加缓存与限频刷新，降低 `OnGUI` 重复事件带来的开销。

### 修复 (Fixed)
* 修复 `RandomLoadout.rules.json` 缺失时只能退回窄范围内置规则的问题，改为优先兜底到全物品随机池模板。
* 修复 `RandomLoadout.rules.json` 存在但解析失败时无法自动切到全物品随机池的问题，并补充对应自动化测试。
* 修复指定与随机输入时因重名显示名导致定位困难的问题，通过别名、导出索引与用户手册强化可发现性与可操作性。
* 修复 Breach 角色页只显示部分初始角色或完全拿不到隐藏角色选项的问题，改为补齐固定角色集合并统一 `Pilot` 显示名。
* 修复隐藏角色在未出现在原生 Breach 角色列表时无法触发解锁的问题，改为允许通过锁定角色按钮直接完成持久解锁。
* 修复角色页顶部状态文案与顶部操作结果提示在长文本时被截断的问题，改为支持自动换行与动态高度。
* 修复 `Release` 模式下首次按 `F7` 打开命令面板出现明显卡顿后才显示 UI 的问题，避免命令页打开时触发角色页的重型查询链路。
